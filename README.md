# pass-keeper

# Запуск бэкенда
Есть два варианта запуска бэкенда:
1. Запуск через docker-compose
    ```shell
    docker-compose up
    ```
2. Запуск командой
    ```shell
    make server_run
    ```
   В данном случае потребуется наличие доступа к БД и указание его в переменной DATABASE_DSN в файле local.env

# Запуск клиента
1. [в зарработке]Скачать релиз последний релиз под необходимую вам ОС
2. Собрать бинарный файлы из исходного кода с помощью команды make build_client.
Бинарный файл будет лежать в дирректории проекта по адресу ./bin/pass-keeper-client
3. Просто запустить сервис без сборки с помощью команда make client_run
